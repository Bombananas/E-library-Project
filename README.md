# E-library-Project
A small collaboration project created for NPIT.

## contribution rule
1. Must make changes and alterations on a separate branch, and name the branch according to the change you want to make.
2. The name of the file must reflect the feature that it does. For example, "editLevel.php", the file name tells its purpose of editing the level in the database.
3. Variable must not be abbreviated without context.
4. The name of a variable must show what it stores. For example, "levelName", the variable name shows that it stores the name of a level.
5. The starting word of a variable must start with a lowercase letter, and then the subsequent word must start with an uppercase letter. For example, "levelName".
6. Function name must not be abbreviated without context.
7. The name of a function must show what it does. For example, "editLevelName", the name of the function shows that it will change the name of the level in the database.
8. The starting word of a Function must start with a lowercase letter, and then the subsequent word must start with an uppercase letter. For example, "editLevelName".
